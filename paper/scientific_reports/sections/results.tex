\subsection{Target sources show a precision--robustness tradeoff}

The default success threshold is 0.08 m, but stored target-error values allow the same results to be recomputed at 0.04 m, 0.06 m, and 0.10 m. Figure~\ref{fig:target-source-threshold-tradeoff} shows that crop-median depth is strongest at the default and relaxed thresholds, whereas box-center depth is more precise under strict 0.04 m and 0.06 m thresholds. We therefore avoid the stronger claim that crop-median depth is universally best.

\subsection{A learned detector plug-in exposes query and adapter limits}

The detector-bridge sweep evaluates metadata-assisted selection, first detection, and one GroundingDINO plug-in through the same RGB-D target-source interface. GroundingDINO produces usable detections but has lower diagnostic success than metadata-assisted selection, and its low 2D IoU does not necessarily imply absence of useful 3D crop evidence. Diagnostic success is computed from lifted 3D target distance, so a detection box can fail a strict 2D IoU criterion while still preserving enough crop depth for a valid target in simple scenes. The low-IoU analysis in Supplementary Table S10 quantifies this effect from rows with recorded IoU fields. The full detector-bridge, query-ablation, and target-name audit tables are provided in Supplementary Tables S6--S9.

\subsection{External real RGB-D validation separates detector and lifting failures}

To test whether the diagnostic logic is only a ManiSkill artifact, we evaluated the same target-generation protocol on the full YCB-V/BOP test split with ground-truth visible masks, visible boxes, camera intrinsics, and object names. This probe does not evaluate robot execution. It uses the visible-mask median as a diagnostic 3D reference target, then compares oracle masks, oracle 2D boxes, first ground-truth boxes, and GroundingDINO detections through the same RGB-D target sources. Oracle-mask lifting reaches 99.97\% success at 8 cm, oracle 2D boxes with crop-trimmed median depth reach 79.9\%, and GroundingDINO improves from 23.8\% with generic prompts to 61.4\% with true object-name prompts (Fig.~\ref{fig:external-rgbd-validation}). Thus, the external probe supports a bounded claim: EmbodiedStressBench separates RGB-D lifting error from detector/query error on real RGB-D data, but it does not establish real-robot manipulation performance. The full external summary and the object-level, scene-level, and detector-threshold stratifications are provided in Supplementary Tables S11--S14.
